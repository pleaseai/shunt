# Product Guidelines

> Auto-generated during `/please:setup` from `AGENTS.md` and `CONTRIBUTING.md`.
> Code style, engineering conventions, and product boundaries for `shunt`.

## Language & Docs

- Write **all** documentation and code in **English** (OSS repo, English-only).
- Code and docs must not drift. When a change alters observable behavior, config keys,
  endpoints, CLI, provider/model support, or defaults, update the affected docs **in the same
  PR** as the code:
  - `README.md` — capability add/remove, setup, supported providers/models, quickstart
  - `docs/` — milestone/spec notes when implementation deviates; new note for a new subsystem
  - `site/src/content/docs/` — user-facing getting-started / guides / reference pages
  - `wiki/` — generated by memex; **never hand-edit**, regenerate separately

## Code Style (Rust)

- Keep Rust files focused and preferably **under 500 lines**.
- **Preserve streaming semantics**: do not buffer upstream SSE responses unless the client
  requested non-streaming output.
- Keep gateway-owned errors in **Anthropic error shape**.
- Prefer **table-driven config additions** over hardcoded provider logic — new providers and
  models should arrive as config, not new code branches.
- Run `cargo fmt`, `cargo clippy` (`-D warnings`), and `cargo test` before reporting a change
  complete.

## Architecture Conventions

- **Routing**: exact route → prefix route → default-provider resolution (`src/routing.rs`).
- **Adapters**: one module per provider protocol under `src/adapters/`; adapters translate,
  they do not own routing policy.
- **Translation**: Anthropic Messages ⇄ OpenAI Responses lives in `src/model/`.
- **Config**: typed, validated, TOML/YAML/env-loaded in `src/config.rs`; add config with
  defaults and validation.
- **Auth**: credential lookup/refresh in `src/auth/`; off-thread store I/O; cancellation-safe
  OAuth refresh.

## Boundaries

- ✅ Always preserve existing tests and add focused coverage for protocol changes.
- ✅ Always run format, clippy, and tests before reporting code changes complete.
- ✅ Always update affected docs (`README.md` / `docs/` / `site/`) in the same PR.
- ⚠️ **Ask** before changing credential-file writeback behavior.
- ⚠️ **Ask** before changing public config keys or documented provider semantics.
- 🚫 **Never** weaken or remove tests to make a change pass.
- 🚫 **Never** commit secrets, tokens, or generated local config files.

## Git & Review

- Use **conventional commits**.
- Keep PRs focused; link milestone docs when changing implementation behavior.
- Third-party GitHub Actions must be pinned to full commit SHAs.
- CI quality gates: SonarCloud `new_coverage` ≥ 80% and `new_duplicated_lines_density` ≤ 3%;
  codecov patch coverage against base. AI reviewers (cubic, greptile enabled) gate auto-merge.
